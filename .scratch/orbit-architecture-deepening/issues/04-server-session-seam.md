## Agent Brief

**Category:** enhancement
**Summary:** Extract a pure server-session module and keep Qt lifecycle behavior behind a thin adapter

**Current behavior:**
Server lifecycle behavior mixes subprocess control, readiness detection, and Qt signaling in one implementation. Tests that only care about start/stop behavior still import GUI-heavy modules, which spreads lifecycle knowledge across the GUI seam and weakens locality.

**Desired behavior:**
A pure server-session module should own subprocess lifecycle, output parsing, readiness detection, and stop/failure behavior. A thin Qt adapter should translate that behavior into the GUI workflow without requiring non-GUI tests to import the full window graph.

**Key interfaces:**
- `ServerSession` (or equivalent) - a non-Qt interface for start, stop, running state, and lifecycle events
- A Qt adapter that exposes the existing GUI-friendly signals or callbacks without owning the core lifecycle logic
- Clear lifecycle states for starting, running, stopped, and failed transitions
- Import structure that allows lifecycle tests to exercise the pure module without importing the whole GUI tree

**Acceptance criteria:**
- [ ] A non-Qt module owns subprocess lifecycle and readiness / failure behavior
- [ ] The GUI still supports the same start, stop, and status-update behavior through a thin adapter
- [ ] Lifecycle-focused tests can import the pure module without pulling in the full GUI graph
- [ ] Existing GUI behavior remains intact at the user-facing seam

**Out of scope:**
- Changing orbit selection or request flow
- Embedding the browser visualization in the desktop application
- Refactoring unrelated GUI widgets
