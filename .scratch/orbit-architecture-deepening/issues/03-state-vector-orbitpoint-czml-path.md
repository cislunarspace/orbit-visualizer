## Agent Brief

**Category:** enhancement
**Summary:** Deepen the state-vector slice so Orbit computation returns OrbitPoint values and CZML formatting owns packet assembly

**Current behavior:**
The state-vector path emits raw transport-shaped dicts, and API handlers assemble CZML packets directly. Time formatting, position shape, and document assembly rules leak across multiple callers, which makes the slice shallow and hard to test through one interface.

**Desired behavior:**
The state-vector slice should produce explicit `OrbitPoint` values from Orbit computation and pass them through one CZML formatting seam that owns document and packet assembly. The state-vector visualization should remain demoable end to end through the API and browser flow.

**Key interfaces:**
- `OrbitPoint` - the shared domain value for time plus 3D position
- A state-vector Orbit computation interface that returns `OrbitPoint` sequences rather than transport dicts
- A CZML formatting interface that accepts named `OrbitPoint` sequences and emits CZML output
- API handlers that delegate to these interfaces instead of rebuilding packet structure themselves

**Acceptance criteria:**
- [ ] The state-vector path returns `OrbitPoint` values instead of raw transport-shaped dicts
- [ ] One CZML formatting interface owns packet and document assembly for the state-vector slice
- [ ] The state-vector branch of the API still renders end to end through the browser-facing CZML flow
- [ ] Tests verify the slice through the Orbit computation and CZML formatting interfaces instead of duplicating handler details

**Out of scope:**
- Moving the TLE path onto the new seam
- Moving broadcast ephemeris or precise ephemeris onto the new seam
- Reworking GUI configuration flows
