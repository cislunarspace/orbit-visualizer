## Agent Brief

**Category:** enhancement
**Summary:** Move the TLE visualization path onto the shared Satellite, Orbit request, OrbitPoint, and CZML formatting seams

**Current behavior:**
The TLE path is wired separately from the emerging deepened slices. It depends on duplicated `Satellite` knowledge, ad hoc request wiring, and handler-owned CZML assumptions instead of reusing the same interfaces as other Orbit computation paths.

**Desired behavior:**
The TLE path should accept a built-in or custom `Satellite`, generate `OrbitPoint` values through the shared Orbit computation seam, and render through the shared CZML formatting seam. It should participate in the same request flow as the state-vector slice.

**Key interfaces:**
- The shared `Satellite` catalog for built-in and default satellite selection
- The shared orbit request interface for selecting a built-in or custom TLE source
- A TLE Orbit computation adapter that returns `OrbitPoint` values
- The shared CZML formatting interface used by other orbit slices

**Acceptance criteria:**
- [ ] The TLE path is implemented behind the shared Orbit computation seam
- [ ] Built-in and custom `Satellite` input both flow through the shared request and catalog interfaces
- [ ] The TLE visualization renders through the shared CZML formatting seam
- [ ] Tests verify a TLE-driven end-to-end slice using the shared interfaces

**Out of scope:**
- Refactoring broadcast ephemeris or precise ephemeris
- Adding live TLE downloads or remote catalogs
- Changing the server lifecycle module
