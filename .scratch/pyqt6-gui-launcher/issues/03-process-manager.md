**Category:** enhancement
**Summary:** Build `ProcessManager` class that wraps uvicorn subprocess with Qt signals

**Current behavior:**
No process management exists; the server is started directly via command line.

**Desired behavior:**
`ProcessManager` (a `QObject` subclass) owns the uvicorn subprocess and exposes `start(host, port)`, `stop()`, and Qt signals for `started(url)`, `stopped`, `failed(error_msg)`, and `output(line)`.

**Key interfaces:**
```python
class ProcessManager(QObject):
    started = Signal(str)   # emits URL string when server is ready
    stopped = Signal()      # emits when server exits after stop()
    failed = Signal(str)     # emits error message on unexpected exit
    output = Signal(str)    # emits each stdout/stderr line

    def start(self, host: str, port: int) -> None: ...
    def stop(self) -> None: ...
    def is_running(self) -> bool: ...
```
- Uses `subprocess.Popen` with `stdout=PIPE, stderr=PIPE, text=True`
- Detects server readiness by scanning for `"Uvicorn running on"` in output
- `start()` runs uvicorn as: `uvicorn main:app --host {host} --port {port}`
- `stop()` calls `process.terminate()` and waits briefly for graceful exit
- Both `start()` and `stop()` are idempotent (no-op if already in desired state)

**Acceptance criteria:**
- [ ] `start("127.0.0.1", 8000)` spawns uvicorn with those arguments
- [ ] `started` emits with `"http://127.0.0.1:8000"` when "Uvicorn running on" is seen

- [ ] `output` emits each line from stdout/stderr in real time
- [ ] `stop()` terminates the subprocess and emits `stopped`
- [ ] `is_running()` returns correct bool in all states
- [ ] `start()` while running is a no-op (no duplicate process)
- [ ] `stop()` while not running is a no-op (no error)
